# Script Outline: Claude Code Agent Teams

> Автор: script-architect
> Статус: DRAFT v2 (research integrated)
> Дата: 2026-03-13

## Тема видео

Claude Code Agent Teams — экспериментальная функция Anthropic, позволяющая запускать несколько специализированных Claude-агентов как единую команду с lead/teammate архитектурой, прямой коммуникацией и параллельным выполнением задач.

---

## Hook-варианты (3 штуки)

### Hook A — "Клонирование"
> "Что если твой AI-ассистент мог бы сам себя клонировать — и каждый клон получал бы свою специализацию? Один ищет информацию, другой проверяет факты, третий пишет сценарий... и все это — одновременно."

### Hook B — "Не один, а команда"
> "Один Claude — это мощно. Но что если вместо одного агента у тебя целая команда? Каждый — со своей ролью, своими задачами, и они реально общаются друг с другом."

### Hook C — "Anthropic сделал неожиданное"
> "Anthropic только что выпустил фичу, которую я не ожидал увидеть ещё минимум год. Они превратили Claude Code из одиночного ассистента — в оркестратор целой команды AI-агентов."

**Рекомендация:** Hook C — сильнее всего для YouTube, создает интригу и устанавливает авторитет темы.

## Структура видео

### 1. HOOK (0:00–0:30)
- **Цель:** Захватить внимание за первые 5 секунд
- **Приём:** Провокационное утверждение + быстрый визуальный тизер работающих агентов в tmux
- **Длительность:** 30 секунд
- **Визуал:** Экран с 6 панелями tmux, все агенты работают параллельно

### 2. ПОЧЕМУ СЕЙЧАС (0:30–1:30)
- **Цель:** Установить контекст и актуальность
- **Ключевые точки:**
  - AI-агенты переходят от single-agent к multi-agent парадигме
  - Anthropic включил Agent Teams как экспериментальную фичу в Claude Code
  - Это первый мейнстримный инструмент, где агенты реально общаются между собой
  - Конкуренция: ни Cursor, ни Copilot пока не имеют аналога
- **Визуал:** Таймлайн развития AI-coding tools

### 3. ЧТО ТАКОЕ AGENT TEAMS (1:30–3:30)
- **Цель:** Объяснить концепцию понятно для нетехнической аудитории
- **Ключевые точки:**
  - Архитектура lead/teammate: один ведущий, остальные — специалисты
  - Каждый агент — отдельная Claude Code сессия со своим контекстом
  - Агенты описаны в `.claude/agents/` как markdown-файлы
  - Общий проектный контекст через CLAUDE.md
  - SendMessage — механизм прямого общения между агентами
  - Параллельное выполнение задач через tmux split-pane
- **Аналогия:** "Как если бы вы наняли команду фрилансеров"
- **Визуал:** Схема архитектуры lead → teammates

### 4. КАК ЭТО РАБОТАЕТ (3:30–6:30)
- **Подсекции:**
  - 4a. Настройка (30 сек) — флаги, tmux, папка agents/
  - 4b. Запуск команды (1 мин) — tmux, инструкция, появление панелей
  - 4c. Параллельная работа (1 мин) — web-researcher и visual-director одновременно
  - 4d. Зависимости (30 сек) — handoff researcher → fact-checker → script-architect
- **Визуал:** Screencast tmux с zoom-in на ключевые моменты

### 5. PROOF-МОМЕНТ (6:30–8:30)
- **Цель:** Доказать, что это работает
- **Показать:** research-summary, verified-claims, video-script, visual-plan, final-report
- **Кульминация:** Side-by-side пустой agent-runtime vs заполненный
- **Визуал:** Файлы + side-by-side сравнение

### 6. ИТОГ + CTA (8:30–9:30)
- **Ключевые точки:** shift к AI-командам, экспериментальная но рабочая, потенциал
- **CTA:** "Какую команду агентов вы бы собрали?"
- **Визуал:** Recap-монтаж + заставка
