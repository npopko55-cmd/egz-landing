# Runtime агентов

Эта папка делает взаимодействие агентов явным и проверяемым.

## Папки

- `shared/`: промежуточные артефакты, которыми пользуются несколько агентов
- `messages/`: handoff-сообщения, блокеры, вопросы и approvals
- `state/`: план, статус-доска и служебные заметки процесса
- `outputs/`: финальные результаты

## Зачем это нужно

Для YouTube-демо невидимое состояние агентов работает слабо. Видимый runtime делает взаимодействие читаемым:

- зритель видит, что получил каждый агент
- handoff-переходы легко показать в кадре
- ошибки и циклы доработки становятся частью истории

## Минимальный протокол runtime

1. Каждый агент перед началом читает текущий brief.
2. Каждый важный результат записывается в файл, а не остается только в чате.
3. Каждый handoff создает сообщение в `messages/`.
4. Каждый блокер создает сообщение с тегом `blocker`.
5. Процесс не считается завершенным, пока `showrunner` не напишет финальную approval-заметку.
